# asr-bench v0.4 — NVIDIA NeMo Engine Design

> **Status:** Approved design spec (brainstorming output). Feeds the implementation plan.
> **Date:** 2026-06-08. **Supersedes scoping:** [`2026-06-07-v0.4-nemo-scoping.md`](./2026-06-07-v0.4-nemo-scoping.md) (research input; this doc records the decisions made from it).
> **Goal:** Add NVIDIA NeMo as asr-bench's 4th `Engine` family, shipping Canary-Qwen-2.5b and Parakeet-TDT-0.6b-v2 as a controlled head-to-head against Whisper on real lecture audio. Local engines only.

## Executive summary

NeMo's **Canary-Qwen-2.5b** tops the Open ASR Leaderboard (5.63% mean WER vs Whisper Large-V3 ~7.4–7.8%), and asr-bench exists to produce exactly this kind of controlled comparison. The hard constraint is identical to WhisperX's: PyTorch ships no CUDA wheels for Python 3.14, and asr-bench's core runs on 3.14 — so NeMo GPU inference **must** run as a subprocess into a dedicated Python 3.12 venv. The integration reuses the proven WhisperX subprocess/runner pattern almost verbatim and is overwhelmingly additive: no new metrics fields, so `schema_version` stays 1 and `render_markdown` + the JSON sidecar stay byte-for-byte stable.

v0.4 ships **two models** plus an ad-hoc escape hatch, **transcription-only**, with **graceful degradation** when the (optional) NeMo venv is absent. The `engines/` package split and PyInstaller bundling are explicitly deferred.

## Decisions (locked in brainstorming)

1. **Both models in v0.4** — `parakeet-tdt-0.6b-v2` (timestamp workhorse, proves the bridge) and `canary-qwen-2.5b` (headline WER).
2. **Canary-Qwen = WER-only, no VTT.** It outputs clean punctuated text but no native timestamps. As a benchmark, the deliverable is the accuracy number (WER/MER/WIL/CER); forced alignment would add a cross-engine dependency and a new code path for zero benchmark value. Canary-Qwen renders as a text-only row (like NIM's `n/a` disk handling). **Parakeet-v2 carries the full caption story** (native word/segment timestamps → VTT + `_Words_<Model>.json`).
3. **Separate `.venv-nemo`** (Python 3.12, own `setup_nemo_venv.ps1`), isolated from `.venv-whisperx`. NeMo's pins (numpy≥2.0 #14505, transformers/huggingface_hub #10940, lightning) are aggressive enough that a shared venv risks one engine's dependency bump silently breaking the other. The cost is ~4GB of duplicated torch+CUDA on disk — accepted in exchange for reproducibility.
4. **Transcription-only — no diarization/DER.** For the two shipped models, neither does diarization; NeMo diarization (Sortformer/clustering) is a *separate model family* with its own pipeline + RTTM ground-truth. WhisperX already owns the diarization story. A NeMo-vs-WhisperX diarization comparison is a clean potential v0.5 item.
5. **Both venv installers are optional and independent.** A user may install neither, one, or both `.venv-whisperx` / `.venv-nemo`. A Whisper/NIM run must never require either. Requesting a `nemo` (or `+whisperx`) model with its venv missing must fail **that model only** with a clear "run the setup script" message — never crash the run or other models.
6. **Registered models + `nemo:<model>` ad-hoc ids.** The two models are first-class named `MODELS` entries; an ad-hoc regex (mirroring `_NIM_ADHOC_RE`) lets users point at arbitrary NeMo models for forward compatibility. The registered two are the validated, first-class set.

**Deferred (out of scope for v0.4):**
- `engines/` package split → separate refactor PR after NeMo is live-validated and merged (keeps NeMo's risky venv bridge localizable; avoids bundling a 2,849-line structural refactor with a feature release). Layout + invariants recorded in the scoping doc.
- PyInstaller binaries → NeMo's 6–10GB subprocess venv can't be frozen anyway; decouple from this release.
- `InProcessNeMo` adapter → would only fire in a ≤3.13 core with NeMo installed (not asr-bench's setup); dead, untested code. Subprocess + Fake only. Trivial to add later.

## nemo_runner.py JSON contract

Standalone script. **All `torch`/`nemo` imports inside functions** (importing the module stays cheap and torch-free).

`build_arg_parser()`: `--audio --model --device --language` + chunk flags (`--chunk-len-secs`, `--max-new-tokens`).

`main()` does the **dual stdout redirect** verbatim from `whisperx_runner.py` (`os.dup2(2, 1)` + `sys.stdout = sys.stderr` during processing; restore fd 1 only to print the final JSON) because NeMo/Lightning/HF are noisy on stdout. Sets `TORCH_FORCE_NO_WEIGHTS_ONLY_LOAD=1` in its env preemptively (torch 2.6+ `weights_only` default breaks some checkpoint loads).

Emits **exactly one** JSON document on stdout:

```json
{
  "text": "<full transcript>",
  "segments": [{"start": 0.0, "end": 4.2, "text": "..."}],
  "words":    [{"word": "...", "start": 0.0, "end": 0.3}],
  "transcribe_sec": 12.34,
  "vram_peak_bytes": 5912345678,
  "language": "en"
}
```

For Canary-Qwen, `segments` and `words` are empty (text-only). `transcribe_sec` times the `.transcribe()`/`.generate()` call itself (excludes model load — more accurate than wall-clocking the whole subprocess). `vram_peak_bytes` from `torch.cuda.max_memory_allocated()`.

**Two branches (the only real divergence):**
- **Parakeet** → `ASRModel.transcribe(timestamps=True)` → `out[0].timestamp['segment']` → VTT cues; `['word']` → word sidecar. Single pass handles up to ~24 min; longer lectures chunk if needed.
- **Canary-Qwen** → 40s **non-overlapping chunked** `SALM.generate()` with `max_new_tokens` raised well above the example's 128 (else transcripts truncate); concatenate chunk texts → full transcript. No timestamps.
- **Ad-hoc `nemo:<model>`** → auto-detect the class (attempt SALM, fall back to `ASRModel`); on unsupported model, exit nonzero with a clear message.

**Audio decode** is owned by the runner and resolved at live-validation: pass the file path straight to NeMo if it accepts arbitrary files; otherwise reuse the NIM path's `decode_to_pcm16` (16kHz mono PCM). The engine is agnostic either way.

## Data flow

Per `nemo` model, inside the existing generic dispatch loop in `main()` (zero changes once registered):

```
resolve_model_entry(id)                         # registered MODELS or nemo:<model> ad-hoc regex
  → ENGINES["nemo"] → NeMoEngine().run(entry, pairs, cfg)
      for each (audio, reference) pair:
        adapter = make_nemo_adapter(cfg)         # SubprocessNeMo (real) or FakeNeMoAdapter (tests)
        result  = adapter.transcribe(audio, nemo_model, device, lang, chunk flags)
            └─ SubprocessNeMo: cmd=[.venv-nemo python, nemo_runner.py, --audio ...]
               (no secrets needed for local NeMo), timeout, check=False
               nonzero → raise RuntimeError(proc.stderr[:500])
               else    → NeMoResult.from_dict(json.loads(proc.stdout))
        # ---- identical to WhisperXEngine.run from here ----
        ref_text = load_reference_text(reference); origin = detect_reference_origin(...)
        norm     = normalize_for_wer(...)
        metrics  = compute_word_metrics(...)        # WER/MER/WIL/CER/S-D-I — one call
        hall     = compute_hallucination_signals(...)  # repeat_coverage, compression_ratio
        if timestamps present (Parakeet):
            write_whisper_vtt(...) + group_words_into_cues(...) + _Words_<Model>.json
        else (Canary-Qwen):
            no VTT; ClipResult marked text-only
        populate ClipResult / ModelResult with engine="nemo"
  → render_markdown(...) + write_results_json(...)   # UNCHANGED
```

**Invariant:** no new `ClipResult`/`ModelResult` fields → `schema_version` stays 1; `render_markdown` + sidecar byte-stable. Canary-Qwen reuses NIM's existing text-only/`n/a` rendering.

## Error handling & graceful degradation

Failures stay **local to the offending model** — never fatal to the run.

| Failure | Detection | Behavior |
|---|---|---|
| `.venv-nemo` missing | `main()` pre-flight via `_default_nemo_python()` (and no `--nemo-python`) | Warn: *"NeMo model `<id>` requested but `.venv-nemo` not found — run `setup_nemo_venv.ps1` or pass `--nemo-python`. Skipping."* Skip NeMo model(s); all other engines run normally. |
| Subprocess nonzero exit (CUDA OOM, load crash, bad id) | `SubprocessNeMo`: `returncode != 0` | Raise `RuntimeError(proc.stderr[:500])`; record model failed, continue to next model. |
| CPU-only torch trap (Windows) | `setup_nemo_venv.ps1` verifies `cuda True` | Script installs cu128 torch FIRST; refuses to finish if `cuda False`. |
| `weights_only` load failure (torch 2.6+) | — | Runner sets `TORCH_FORCE_NO_WEIGHTS_ONLY_LOAD=1`. |
| Noisy stdout breaks `json.loads` | — | Dual redirect in runner `main()`. |
| Canary-Qwen chunk truncation | live-validation on a ~50-min lecture | `max_new_tokens` raised per chunk; chunk-boundary WER spot-checked before merge. |
| Bad ad-hoc `nemo:<model>` | runner class auto-detect fails | Runner exits nonzero with clear message → per-model-skip path. |
| Secrets in sidecar | `_config_to_dict` | `nemo_python` is a path, not a secret → included for reproducibility; no NeMo secrets exist (local checkpoints). |

## Build sequence (mirrors WhisperX shipping precedent)

1. **Pin the version triple** — install-test on Windows 11 / RTX 5090; find a `(Python 3.12, torch+cu128, nemo_toolkit)` combo loading both `canary-qwen-2.5b` (SALM) and `parakeet-tdt-0.6b-v2` (ASRModel, `timestamps=True`) without resolver conflicts. Record it.
2. **`setup_nemo_venv.ps1`** — cu128 torch FIRST, then `nemo_toolkit[asr]`, verify `cuda True` + tiny model load.
3. **`nemo_runner.py`** — lazy heavy imports, `build_arg_parser`, branch on model class, pure-JSON-stdout main with dual redirect, `TORCH_FORCE_NO_WEIGHTS_ONLY_LOAD=1`, emit the JSON contract.
4. **`NeMoEngine` + adapters in `asr_bench.py`** — `NeMoResult`; `NeMoAdapter`/`SubprocessNeMo`/`FakeNeMoAdapter`; `make_nemo_adapter` + `_default_nemo_python`; `NeMoEngine.run` cloned from `WhisperXEngine.run`.
5. **Register** — MODELS entries; `ENGINES["nemo"]=NeMoEngine`; `_MODEL_VRAM_COST`; `nemo:<model>` ad-hoc regex.
6. **Thread config** — `RunConfig.nemo_python`, `--nemo-python` flag, `cfg=RunConfig(...)` wiring, `_config_to_dict`, `main()` pre-flight check.
7. **Tests** — contract update; `FakeNeMoAdapter` engine tests; packaging entry; keep core torch-free.
8. **Live-validate on 5090** — cu128 wheels + `cuda True`; measure VRAM; Canary-Qwen chunked vs Whisper Large-V3-Turbo on the 12-lecture corpus.
9. **Docs + decision log** — validated triple, captions decision, "NeMo added in-file; split deferred" in CLAUDE.md **and** AGENTS.md mirror (cross-agent sync rule), plus SPEC.md.

## Risks & mitigations

| Risk | Mitigation |
|---|---|
| Python 3.14 GPU hard blocker (no torch cp314 CUDA wheels) | Subprocess to `.venv-nemo` (Python 3.12) is mandatory. |
| CPU-only torch trap on Windows | `setup_nemo_venv.ps1` installs cu128 torch FIRST; verify `cuda True`. |
| Dependency collisions (numpy≥2.0, transformers/hf, lightning) | Dedicated venv; never share with core or `.venv-whisperx`. Pin a known-good triple. |
| Canary-Qwen no native timestamps | WER-only/no-VTT (decided); Parakeet-v2 carries captions. |
| 40s encoder window + truncation | Chunked inference (40s non-overlapping); raise `max_new_tokens`; validate on a real ~50-min lecture. |
| Two divergent APIs (SALM vs ASRModel) | Runner branches on model class; budget for two paths in one runner. |
| Pure-JSON stdout fragility | Replicate `whisperx_runner.py` dual redirect verbatim. |
| `weights_only` checkpoint-load failures (torch 2.6+) | `TORCH_FORCE_NO_WEIGHTS_ONLY_LOAD=1` in subprocess env. |
| Undocumented VRAM for Canary-Qwen | Measure live on 5090 via NVML before merge; populate `_MODEL_VRAM_COST`. |
| Missing venv crashing the run | Graceful per-model skip with clear setup message (pre-flight). |
| `_config_to_dict` omission | Add `nemo_python` (non-secret) so reproducibility command/sidecar is complete. |
| "Local engines only through v0.4" rule | Default `NeMoEngine` to a local in-venv checkpoint; no hosted NeMo NIM/HTTP path. |

## Open items carried into implementation (resolved at build/live-validation, not blockers)

- Exact `(Python 3.12, torch+cu128, nemo_toolkit)` triple — needs a real install test; record once green.
- Whether NeMo accepts arbitrary audio files or needs `decode_to_pcm16` — runner owns this; determined when the venv is up.
- Whether to pin against PyPI `nemo_toolkit` 2.7.3 now or wait for the June-2026 post-split NeMo Speech release — default: pin against the current known-good release; re-pin only if a blocker appears.
