# v0.5 — Community Models (Conformer + distil-whisper) Design

**Date:** 2026-06-08
**Status:** Approved (brainstorm) — ready for implementation planning
**Milestone:** v0.5 (per the SPEC.md roadmap line: *"Stretch. Wav2vec2-large, conformer open models, distil-whisper community fine-tunes. Criteria: locally runnable, Python wrapper, timed segments output."*)

## Goal

Add distil-whisper and HuggingFace-`transformers` community ASR models to asr-bench, and land the long-deferred `engines/` package split as the structural foundation for the fifth engine family.

## Scope

One spec, **three sequenced phases**:

- **Phase 0** — `engines/` package split (pure refactor, zero behavior change).
- **Phase 1** — `distil-large-v3.5` via the existing faster-whisper engine (config-level add).
- **Phase 2** — a new HuggingFace-`transformers` engine (`wav2vec2-large-960h` + `wav2vec2-conformer-large`) in its own `.venv-hf`.

Phases 0 and 1 are fully offline-TDD-testable on any box. Phase 2 carries a **live-validation gate on the RTX 5090 before merge**, identical to the NeMo and WhisperX milestones.

### Models targeted (locked)

| Model id | Phase | Engine | Timestamps / output |
|---|---|---|---|
| `distil-large-v3.5` | 1 | faster-whisper (existing) | native Whisper timestamps → full VTT |
| `wav2vec2-large-960h` | 2 | hf (new) | CTC frame→word timestamps → full VTT + `_Words_*.json` |
| `wav2vec2-conformer-large` | 2 | hf (new) | CTC frame→word timestamps → full VTT + `_Words_*.json` |

`distil-large-v3` (older distil) was considered and **cut** — `distil-large-v3.5` supersedes it for this domain. wav2vec2/Conformer cover the roadmap's "wav2vec2-large" and "conformer open models"; distil-large-v3.5 covers "distil-whisper community fine-tunes."

## Invariants (hold across all three phases)

1. **`import asr_bench` and `python asr_bench.py` stay byte-stable** in public surface — every existing test imports unchanged.
2. **Core stays torch-free** — `import asr_bench` must not pull torch (guarded by `tests/test_packaging.py::test_core_dependencies_are_minimal_and_torch_free`). Heavy imports (torch, transformers, nemo, whisperx) live inside engine `run()` methods and the standalone runners, never at module top.
3. **`schema_version` stays 1** — additive only; no new `ClipResult`/`ModelResult` fields. `render_markdown` and the JSON sidecar are byte-stable for existing engines.
4. **Local engines only** (anti-goal: cloud APIs). All four targeted models run locally.
5. **Static files only** (anti-goal: streaming).

## Phase 0 — `engines/` package split

### Why now

Phase 2 is the **fifth** engine family. The decision log pre-committed the split to "the next engine family" twice (at NeMo, then deferred again). Adding a fifth `Engine` subclass to the ~2,750-line `asr_bench.py` is the textbook "file doing too much" smell. Doing the split now, with the full 263-test suite as a safety net, is safer than after the file grows further.

### Structure (Approach A — shared `base.py`, re-exported from `asr_bench.py`)

```
engines/
  __init__.py        # assembles ENGINES registry; re-exports Engine
  base.py            # Engine ABC + shared torch-free primitives
  faster_whisper.py  # FasterWhisperEngine
  nim.py             # NimEngine + NIM helpers
  whisperx.py        # WhisperXEngine + WhisperXResult + adapters + writers
  nemo.py            # NeMoEngine + NeMoResult + adapters
```

**`engines/base.py`** (torch-free) holds everything ≥2 engines reference:
- `Engine` ABC
- `ClipResult`, `ModelResult`, `RunConfig`, `Pair` (data only)
- `WordMetrics`, `compute_word_metrics`, `compute_hallucination_signals`, `normalize_for_wer`, `_repeat_coverage`, `_compression_ratio`
- `VramSampler`, `decode_to_pcm16`, `group_words_into_cues`
- `write_whisper_vtt`, `_fmt_vtt_time`, `_model_label`, `_audio_duration_sec`

**`engines/__init__.py`** imports each engine module and builds:
```python
ENGINES = {
    "faster-whisper": FasterWhisperEngine,
    "nim": NimEngine,
    "whisperx": WhisperXEngine,
    "nemo": NeMoEngine,
    # "hf": HFTransformersEngine  # added in Phase 2
}
```

**`asr_bench.py`** keeps: NVIDIA DLL setup, `MODELS`, `resolve_model_entry`, `_MODEL_VRAM_COST`, `recommend_batch_size`, corpus discovery (`discover_pairs`, reference loading, caption parsing), `render_markdown`, JSON sidecar builders, the entire fusion stage, LLM backends, `prepare_gold`, and `main()`. It **re-exports** the relocated names so the public surface is unchanged:
```python
from engines.base import (
    Engine, ClipResult, ModelResult, RunConfig, Pair, WordMetrics,
    compute_word_metrics, compute_hallucination_signals, normalize_for_wer,
    VramSampler, decode_to_pcm16, group_words_into_cues,
    write_whisper_vtt, _model_label, _audio_duration_sec,
)
from engines import ENGINES
```

### Dependency DAG (acyclic — no circular imports)

```
engines/base.py   →  (stdlib + jiwer only; torch-free)
engines/<eng>.py  →  engines/base
engines/__init__  →  engines/<eng>.py
asr_bench.py      →  engines, engines.base
```

`asr_bench` no longer needs the engines to import back from it — the shared primitives moved down into `base`. This is the reason Approach A was chosen over "engines import from asr_bench" (which is circular and only works via fragile late imports).

### Packaging

`pyproject.toml` adds the `engines` package to the build alongside the existing flat `py-modules` (`asr_bench`, `asr_compare`, `nemo_runner`, `whisperx_runner`, and `hf_runner` in Phase 2). `test_packaging.py` is extended to assert `engines` and its submodules are included and that importing `asr_bench` (which now imports `engines`) stays torch-free.

### Execution discipline

- Pure mechanical extraction; **no behavior change**.
- Extract one engine module per commit; **run the full test suite (263 tests) after each extraction**.
- The only new tests are import-surface assertions: the re-exported names resolve from `asr_bench`, and `ENGINES` contains the same keys as before.

## Phase 1 — distil-large-v3.5 (faster-whisper config)

### Changes

- Add to `MODELS`:
  ```python
  "distil-large-v3.5": {
      "engine": "faster-whisper",
      "fw_name": "<CTranslate2 distil-large-v3.5 id>",  # pinned in the plan after verifying availability
      "params": "756M", "label": "Distil-Whisper Large V3.5",
      "notes": "Community distil fine-tune of large-v3. English-only.",
  },
  ```
  The exact `fw_name` (a CTranslate2 conversion id — e.g. a `distil-whisper/...` or `Systran/faster-distil-whisper-...` HF id) is verified for availability and pinned in the implementation plan, not guessed here.
- Add a `_MODEL_VRAM_COST` entry for `batch_size auto` sizing.

### Behavior

Runs through the **existing** `FasterWhisperEngine` path unchanged — native Whisper timestamps → full VTT. No new engine code. English-only, which suits the lecture domain.

### Tests

- `resolve_model_entry("distil-large-v3.5")` returns an entry with `engine == "faster-whisper"`.
- `_MODEL_VRAM_COST` has an entry for the new id.

## Phase 2 — HF-transformers engine (wav2vec2 + Conformer)

### Components

**`engines/hf.py`** — `HFTransformersEngine(Engine)` plus the subprocess adapter set, mirroring `nemo.py`:
- `HFAdapter` (ABC), `FakeHFAdapter` (for tests), `SubprocessHF` (spawns the runner in `.venv-hf`)
- `make_hf_adapter(cfg)`, `_default_hf_python()` (auto-detects `./.venv-hf`), `_hf_runner_args(...)`
- `ENGINES["hf"] = HFTransformersEngine`

**`hf_runner.py`** (new top-level py-module, runs inside `.venv-hf`):
- Uses `transformers.pipeline("automatic-speech-recognition", model=<id>, chunk_length_s=30, stride_length_s=5, return_timestamps="word", device=0)`.
- Built-in chunking + stride handles long lectures (no manual windowing, unlike Canary-Qwen).
- CTC frame→word timestamps → `words` + `segments` (grouped via the shared `group_words_into_cues`) → **full VTT + `_Words_*.json`** for both models.
- Pure-JSON stdout; dual `stdout`→`stderr` redirect (NeMo/WhisperX pattern).
- ffmpeg decode-to-16kHz-mono-WAV front-stage for any non-wav input (mirrors `nemo_runner._ffmpeg_to_wav` / `_needs_decode`).
- Heavy imports (torch, transformers) inside the run function; torch-free helpers (arg parser, timestamp shaping, model-type detection) at module scope for unit testing.

**`setup_hf_venv.ps1`** — provisions `.venv-hf`:
- Python 3.12 (torch has no 3.14 wheels).
- **cu128 torch FIRST**, then `transformers` + `torchaudio` + `soundfile` (avoids the CPU-only-torch trap).
- Verifies `torch.cuda.is_available()`; hard-fails if False.
- **Pure ASCII** from the start (the PowerShell-5.1 em-dash encoding lesson, baked in).

### Registration & CLI

- `MODELS` gains `wav2vec2-large-960h` and `wav2vec2-conformer-large` (`"engine": "hf"`), each with a `_MODEL_VRAM_COST` entry.
- `hf:<model>` ad-hoc IDs in `resolve_model_entry` via a regex (`_HF_ADHOC_RE`), mirroring `nemo:`/`nim:`.
- New CLI flag `--hf-python` (path to the venv python; auto-detects `./.venv-hf`).
- `hf_python` added to the sidecar config (non-secret).

### Graceful degradation

If a `hf`-engine model is requested without `.venv-hf` present, `main()`'s pre-flight **skips just those models with a warning** and runs the rest of the requested engines — identical to the NeMo handling. A bare Whisper run never needs `.venv-hf`.

### Outputs

Both models produce a full VTT + `_Words_*.json` sidecar (CTC gives word timestamps). No WER-only special-case like Canary-Qwen.

### Tests

- `hf_runner.py` torch-free helpers: arg parser defaults/required, timestamp/segment shaping from dict + object inputs, `_needs_decode`, model-type detection.
- `HFTransformersEngine` via `FakeHFAdapter` (mirrors `tests/test_nemo_engine.py`): produces a `ModelResult` with the right metrics, VTT + words sidecar written, graceful skip when no venv.
- `.gitignore` gains `.venv-hf/`.

### Live-validation gate (RTX 5090, before merge)

Download `wav2vec2-large-960h` and `wav2vec2-conformer-large`, run `hf_runner.py` on a real mp4 lecture, then a full `asr_bench` run alongside Whisper/NeMo. Record the version triple (Python · torch · transformers), peak VRAM, and first-run WER/RTFx into CLAUDE.md/SPEC.md. Fix whatever the live run surfaces (the gate exists precisely for bugs unit tests cannot catch). Merge only after green.

## Error handling summary

| Failure | Handling |
|---|---|
| `.venv-hf` missing, hf model requested | Pre-flight skip with warning; other engines run |
| Non-wav input (mp4/m4a) | ffmpeg decode-to-wav front-stage in `hf_runner.py` |
| ffmpeg missing | Clear `RuntimeError` with install hint (mirrors NeMo) |
| Runner subprocess non-JSON chatter on stdout | Dual stdout→stderr redirect; only final JSON on real stdout |
| CPU-only torch in `.venv-hf` | `setup_hf_venv.ps1` installs cu128 first + verifies CUDA, hard-fails otherwise |
| distil `fw_name` unavailable as CT2 | Caught during the implementation plan (pin a verified id); not deferred to runtime |

## Testing strategy summary

- **Phase 0:** existing 263 tests stay green after every extraction commit + new import-surface assertions. No behavior change.
- **Phase 1:** registration + VRAM-cost tests (offline).
- **Phase 2:** torch-free runner-helper unit tests + fake-adapter engine tests (offline) **plus** the RTX 5090 live gate before merge.
